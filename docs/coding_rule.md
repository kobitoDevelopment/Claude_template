# コーディングルール

## 共通

### ロジック作成指針

DRY 原則に則り、共通化できるロジックは共通化して再利用する。

### コンポーネント作成指針

1 つのコンポーネントに 1 つの役割とし、関心の分離を適切に実施することで、コンポーネントの責務範囲を明確にし、テストの独立性を確保、再利用性・保守性・可読性を向上させる。

### コメント規約

コンポーネントやロジックを作成するとき、必ず JSDoc 形式でそのファイルを説明するコメントを追記する。

### UI とロジックの分離

コンポーネント作成時は必ず UI に関するファイルとロジックに関するファイルを分離し、ロジックは UI に関するファイルから import する。

## TypeScript

- interface は使用せず、type による型定義を行う。
- class 構文は使用せず、関数型プログラミングを行う。

## React

- React のライフサイクル・作法に則ってロジックを記述する。(例: 適切に状態管理すべき箇所で let を使用した再代入などを行わない)

## PHP

- 型注釈を必ず記述する。
