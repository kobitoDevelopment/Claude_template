# CLAUDE.md

## 使用する言語

必ず日本語を使用してください。文字化けに注意。文字コードは UTF-8 を使用。

## 命令を受けてからコードを編集するまでの流れ

1. ユーザーからの命令を受け取る
2. 受け取った命令を理解し、何を達成すべきかユーザーに確認する
3. 達成すべき目的のために作成・編集・削除するファイルを、根拠を添えて列挙し、ユーザーに確認する
4. ユーザーからの確認を受け取ったら、命令の目的を達成するための詳細な実行計画をユーザーに提示する
5. ユーザーからの確認を受け取ったら、実行計画に基づいてコードを編集する

## コーディングルール

- 作業時は必ず docs/coding-rule.md のルールを遵守してください。
- コメントを記述するときは、客観的に事実に基づく表現のみを使用してください。「重要な」「完全な」などの形容詞はノイズでしかありません。主観的で曖昧な表現は絶対に混入させてはいけません。

## 実行時の厳守事項

- **影響範囲の確認**: 意図しない変更や影響が発生する箇所を特定して作業を実施する
- **曖昧さの回避**: 「~の可能性があります」「〜かもしれません」を禁句とし、必ず一次情報を根拠に作業してください。根拠となるソースの URL と該当箇所の引用を必ずユーザーに知らせること。
- **指示にないスタイリングの追加**: 独断で UI のデザインを変更、追加しないこと。
- **エラーの隠蔽**:失敗しているテストを skip したり、削除したり、エラーを隠蔽したりするような行為は行わないこと。
